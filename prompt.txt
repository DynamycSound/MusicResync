## What You're Building

A desktop lyrics syncing app (Windows primary, cross-platform if easy) that:
1. Takes a folder of music files (even with **bad/incomplete metadata** — artist names wrong, missing tags, etc.)
2. Finds and downloads **synced lyrics (.lrc files)** for every song from multiple providers
3. Has a **confidence scoring system** to validate matches
4. Has a **manual fallback UI** for unmatched/low-confidence songs
5. Has an **auto mode** that does best-effort matching after the main batch
6. Organizes songs into tabs: **All / Has Lyrics / No Lyrics**
7. Samsung Music compatible (`.lrc` files named identically to the audio file)

NOTE: Per user direction during implementation, this was built as an ANDROID app (fork of SongSync, Kotlin/Compose),
not a desktop app. See README.md and the plan for details.

## Goals
- Fix all providers (Netease, Apple Music, Musixmatch, Deezer, LRCLib, Spotify) with rate-limit handling.
- Bad metadata handling: multi-strategy search (metadata, filename parsing, fuzzy, duration match, optional fingerprinting).
- Confidence scoring (title 40 / artist 30 / duration 20 / album 10; thresholds 0.85 / 0.60).
- Tabs: All / Has Lyrics / No Lyrics.
- Auto mode + manual search UI.
- Samsung Music compatible .lrc output (identical basename next to audio).
- Scan for _private suffix .lrc and strip it so players see them; detect already-correct .lrc.
- "Correct the Metadata" pre-batch checkbox (off by default).
- Keep GPL-3.0 license (derivative work).
